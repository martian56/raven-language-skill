# rvpm and the `raven` CLI

`rvpm` is the Raven package manager / project runner. `raven` is the interpreter binary. Both must be on `PATH` for `rvpm run` to work (rvpm shells out to `raven`).

## Project layout

```
my_project/
├── rv.toml          # project manifest
├── .gitignore       # autogenerated by `rvpm init` — ignores rv_env/packages/
├── src/
│   └── main.rv      # entry point
└── rv_env/
    └── packages/    # reserved for future installed packages
```

## `rv.toml`

Default written by `rvpm init`:

```toml
[package]
name = "my_project"
version = "0.1.0"
authors = []

[dependencies]
# Add packages here, e.g.:
# math = "1.0"

# [fmt]
# indent_width = 4
# wrap_width = 88
```

Sections:

- `[package]` — project metadata.
- `[dependencies]` — declared but `rvpm install` / `rvpm add` are **not yet implemented** as of v1.4.x. Don't promise the user that adding entries here will pull anything down.
- `[fmt]` — optional, controls the formatter.
  - `indent_width` (default `4`)
  - `wrap_width` (default `88`)

## Subcommands

### `rvpm init [name]`

Scaffolds a new project. With `name` omitted (or `.`), uses the current directory and refuses to clobber an existing `rv.toml`. With a name, creates the directory if it doesn't already contain entries.

Writes:
- `rv.toml`
- `src/main.rv` (a "Hello from Raven!" stub that calls `main()`)
- `rv_env/packages/`
- `.gitignore` (only if it doesn't already exist)

### `rvpm run`

Walks up from the current directory to find the nearest `rv.toml`, then runs `raven src/main.rv` from that directory. Exits with the interpreter's exit code.

Failure modes:
- `rv.toml` not found anywhere on the way up → "Not a Raven project" error.
- `src/main.rv` missing → tells you to `rvpm init`.
- `raven` not on `PATH` → "Failed to run raven: ... Is 'raven' in PATH?"

### `rvpm fmt [paths…]`

Formats Raven source in place. With no paths, formats `src/` of the current project. With paths, formats each (file or directory).

Flags:
- `--check` — exits non-zero if anything would change. Use in CI.

Reads `[fmt]` section of the nearest `rv.toml` if present.

### `rvpm install`, `rvpm add` 

Declared but not implemented. They print "not yet implemented" and exit 1.

## The `raven` CLI

The interpreter itself.

```bash
raven file.rv          # parse, type-check, execute
raven file.rv -c       # parse and type-check only — does NOT execute
raven file.rv -v       # verbose: dumps the AST etc.
raven                  # REPL
```

The REPL handles multi-line input by tracking `(`, `{`, `[` depth (and ignoring delimiters inside strings / comments). When depth returns to zero, it parses and runs the buffered text.

### `-c` is your friend

When iterating on Raven code, `raven file.rv -c` is the fastest way to surface parse and type errors. It's much faster than running, and it catches the bulk of beginner mistakes (wrong type, missing annotation, unknown function name).

## Module resolution

`import` paths resolve in this order (see `src/paths.rs`):

1. **Current working directory** (where you ran `raven`).
2. **`RAVEN_LIB_PATH`** environment variable (colon-separated on Unix, semicolon on Windows).
3. **Relative to the `raven` binary** — typically `<install>/lib/`.
4. **System paths** (Linux: `/usr/lib/raven`, etc.).

For `import "foo"` (quoted): tries `foo.rv` in each search root.
For `import foo` (bare, stdlib-style): same lookup; the stdlib lives next to the binary.

If you install Raven via the prebuilt installer, the stdlib is bundled and resolved automatically. If you're running from source, `cargo run --release -- file.rv` works because the resolver walks back to the workspace root.

## Common workflows

### Start a project

```bash
rvpm init my_thing
cd my_thing
rvpm run
```

### Add a module

Drop another `.rv` file into `src/`, e.g. `src/util.rv` with `export fun ...`. Then in `src/main.rv`:

```raven
import "util";
```

No build step — the interpreter discovers it on import.

### Type-check before running

```bash
raven src/main.rv -c
```

If clean, run with `rvpm run`.

### Format

```bash
rvpm fmt
rvpm fmt --check     # CI verification
```

### Install Raven for a fresh user

Get the `raven` binary onto `PATH` (build it with `cargo build --release` from the compiler repo and copy `target/release/raven`, or use a packaged installer). Optionally also install `rvpm` (the package manager binary) the same way. Then `raven --version` should work and `rvpm init` should scaffold a project.
